# Standalone Simple Example

This example demonstrates the **standalone mode** of Zngur, which generates a complete bridge project instead of individual files.

## Key Differences from Regular Mode

1. **Automatic compilation**: `zngur gs` auto-builds your crate if needed (no manual `cargo build` required)
2. **Clean separation**: User code (`src/lib.rs`) has no references to generated code
3. **Auto layout works seamlessly**: Using `#layout(auto)` throughout `main.zng`
4. **Git-friendly**: The entire `zngur-bridge/` directory is in `.gitignore`
5. **Single library output**: Only need to link against `-lzngur_bridge` (includes everything)

## Build Steps

```bash
make clean   # Clean all generated files
make         # Build everything
make test    # Run and verify output
```

## How It Works

The Makefile orchestrates the build in just 3 simple steps:

1. **Generate bridge project** (`zngur gs main.zng`) - Automatically builds your crate if needed for `#layout(auto)`
2. **Build bridge crate** (`cd zngur-bridge && cargo build --release`) - Cargo builds your crate as dependency
3. **Compile C++** (`clang++ -lzngur_bridge`) - Link only against the bridge (includes everything!)

## Generated Structure

```
standalone-simple/
├── zngur-bridge/          ← Generated by zngur gs
│   ├── Cargo.toml         (depends on example-standalone-simple)
│   ├── build.rs           (compiles C++ FFI code)
│   ├── src/lib.rs         (FFI + layout assertions)
│   └── cpp/
│       ├── generated.h
│       └── generated.cpp
└── ...
```

## Advantages

- ✅ Works with `#layout(auto)` without any hacks
- ✅ No `mod generated;` in user code
- ✅ Clear dependency order
- ✅ Easier to debug (separate compilation units)
- ✅ Git-friendly (ignore entire bridge dir)
